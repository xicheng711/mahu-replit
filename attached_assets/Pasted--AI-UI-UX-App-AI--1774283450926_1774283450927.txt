核心功能重构：从“AI建议驱动”转向“数据可视化记录+温馨陪伴”
你好！我们需要对当前项目进行一次重大的产品逻辑和 UI/UX 重构。App 的核心定位将从“生成长篇 AI 护理建议”转变为“便捷的数据记录 + 美观的数据可视化简报 + 温馨的 AI 陪伴”。

请严格按照以下 5 个阶段逐步修改代码。在修改前，请先仔细阅读所有要求。并在修改过程中，确保不要破坏现有的 Expo Router 路由结构。

阶段 1：数据库 Schema 调整 (Drizzle ORM)
我们需要修改打卡（Check-in）和日记（Diary）的数据表结构。

修改 Check-in 表 (Morning Check-in)

新增字段 sleep_type: string (枚举: 'quick' | 'detailed')，记录用户使用的是快捷输入还是详细分段输入。

新增字段 sleep_segments: json (或 text 存 JSON 字符串)，用于存储详细的睡眠时间段。注意：必须存储包含日期的完整 ISO 格式（例如 [{ start: "2026-03-22T23:00:00Z", end: "2026-03-23T06:30:00Z" }]），以正确处理跨越午夜的计算。

新增字段 night_wakings: integer，记录夜里醒来的次数。

新增字段 daytime_naps: boolean (或 string)，记录白天是否有小睡。

移除或废弃字段: 早上打卡不再记录照顾者心情（caregiver_mood 相关字段从早上打卡逻辑中移除）。

修改 Diary 表 (日记)

新增字段 caregiver_mood: string (或 emoji)，将照顾者的心情记录转移到日记表中。

阶段 2：后端 tRPC & AI Router 调整
为了让生成速度变快，我们需要大幅精简 AI 的任务，并增加图表所需的数据接口。

重构 aiRouter.getDailyAdvice (或对应的简报生成接口)

大幅精简 Prompt：彻底移除生成 adviceCards（护理建议）和 nutritionAdvice（营养建议）的要求。

新的 AI 任务：只需要 AI 根据昨晚和今早的数据，输出极短的 JSON：

careScore: 护理综合分数（1-100）。

summary: 一句客观的昨日情况小结。

encouragement: 一句给照顾者的温暖鼓励（20字以内）。

确保生成速度达到秒级响应。

新增接口 getWeeklySleepData

编写一个新的 tRPC query，获取该老人过去 7 天的睡眠数据（总时长、醒来次数、具体的 sleep_segments），用于前端渲染趋势图表。

阶段 3：前端 UI - 早上打卡页面 (Morning Check-in)
重构早上打卡的流程和 UI，注重长辈/照护场景的易操作性。

睡眠记录模块（核心交互）

提供一个 Toggle/Tab 切换：【快捷记录】 vs 【详细记录】。

快捷记录：保留现有的滑动选择器（Slider 或 Picker），直接选择总睡眠时长（如 6.5 小时）。

详细记录：提供一个列表，用户可以点击“添加睡眠时间段”。每个时间段包含 Start Time 和 End Time（使用原生的时间选择器 DateTimePicker）。顶部实时自动计算并显示这些时间段的总时长。

同页面追加问题：

“夜里醒了几次？”（提供 - 0 + 的步进器 Counter）。

“白天有小睡吗？”（Yes/No Toggle）。

移除与保留

移除：早上打卡中关于“照顾者今日心情”的询问步骤。

保留：“还有什么要补充的吗？”（Notes 文本区域）。

阶段 4：前端 UI - 日记功能 (Diary)
将心情记录整合进日记，并强化日历的视觉反馈。

写日记页面

在输入日记正文的上方，增加一个“今日心情”选择器（提供几个代表心情的 Emoji 或 Icon 供单选，必填项）。

日记日历组件 (Calendar)

修改现有的日历组件（如 react-native-calendars）。

在日历的日期格子上，直接显示该天记录的心情 Emoji/Icon。性能要求：请务必使用 useMemo 处理自定义日期标记的渲染，避免滑动月份时发生卡顿。

阶段 5：前端 UI - 全新“护理简报”页面 (Beautiful Summary)
这是本次重构的视觉重点。打卡完成后，展示一个精美的、可分享的简报页面。

引入图表库

请安装 react-native-gifted-charts。

简报页面排版设计 (Top to Bottom)

UI 风格：现代、温馨、留白充足，使用带柔和阴影的圆角卡片（Tailwind/NativeWind）。

头部：护理分数（大字号） + AI 生成的“温暖的鼓励”。

昨日回顾模块 (Yesterday's Summary)：

横向排列的三个 Icon 卡片：昨日被照护者心情、用药准时度、吃饭情况。

昨日睡眠可视化 (Yesterday's Sleep Visual)：

如果用户输入了详细时间段：用一条水平时间轴（Timeline）或分段进度条展示具体的入睡和醒来区间。

如果是快捷输入：用一个美观的环形图（Ring Chart）展示总睡眠时长。

近一周睡眠趋势 (Weekly Sleep Trend)：

使用柱状图（Bar Chart）。X 轴是日期，Y 轴是睡眠时长。在柱子上方或内部用明显标识（如红点）标出“夜间醒来次数”。数据要求：如果某天没有打卡数据，图表 X 轴不能断裂，该天柱状图高度显示为 0 或显示灰色占位。

底部动作区：

系统自动将此简报数据同步至“家庭共享”状态流（需触发数据重新获取）。

保留现有的 react-native-view-shot 截图逻辑，提供“保存长图分享到微信”的显眼按钮。
